# Aguacate — project principles

**Core product principle: zero manual labor.**
The user only records/talks. Everything else is auto-generated by Aguacate from the
conversation — the meeting **title**, summary, actions, decisions, topics, questions,
timeline, and highlights. Never add UI that asks the user to type a title, tag, label,
or manually organize. Titles/knowledge only appear *after* processing (e.g. the capture
flow reveals the meeting name at the Ready phase, not during recording).

## Design language (locked, from user's mockup)
- Calm, mostly white; **green (#7DBE3A / text #6FA83C·#5F9E2B) is the only brand accent**.
- Editorial serif (Newsreader) for big titles + wordmark; **Hanken Grotesk** for UI sans; JetBrains Mono for timestamps/meta.
- Icons: one system — 24 grid, 1.75 stroke, round caps + joins (rounded-flat family).
- The logo = avocado outline with a **black (#111111) waveform** inside (matches uploads/logo-primary.svg). Waveform animates while recording.
- Rounded corners, flat, generous whitespace. Avoid heavy tints/gradients.

## Structure
- `Aguacate Meeting.dc.html` — main canvas: 5g capture flow, 5a Overview, 5c Timeline, 5d Transcript, 5e Ask, 5f Home, 5b moments.
- `AguacateChrome.dc.html` — shared sidebar + meeting list (imported by the app screens).
- `CaptureFlow.dc.html` — animated capture → grow → ready prototype.
